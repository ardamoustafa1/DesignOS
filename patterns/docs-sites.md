# Documentation Sites

Docs are evaluated before the product is (`industries/analytics-hosting.md`). For dev
tools they're also the highest-traffic surface and the real onboarding. Design docs as
a product with one metric: **time-to-working-answer.**

## The three-panel shell
```
Left    sidebar nav 260–280px · the CONTENT MAP (see IA below) · sticky, own scroll
Center  article 680–760px measure · the reading column
Right   "On this page" TOC 200–240px · h2/h3 anchors · scrollspy active state ·
        collapses first on narrow screens
```
Top bar: logo + docs-scoped search (⌘K — `components/search-command.md`; search is the
#1 docs navigation), version switcher, links back to main site/dashboard.

## Information architecture (the Diátaxis-shaped truth)
Four content types users arrive needing — never blend them in one page:
| Type | Question | Shape |
|---|---|---|
| **Quickstart/tutorial** | "get me to hello-world" | numbered steps, ONE happy path, ≤10 min, zero theory |
| **How-to guides** | "how do I do X" | task-named pages, steps + variations |
| **Reference** | "what are the exact params" | generated-feeling tables, exhaustive, dry |
| **Concepts** | "how does this work" | prose + diagrams, why-shaped |

Sidebar groups in that order: Getting started → Guides → Reference → Concepts (+ Changelog,
`patterns/changelog.md`). The quickstart is the most-designed page in the product's world.

## The article
- Type discipline at maximum (`foundations/typography.md`): 16–17px body, 1.7 line-height,
  h2 with generous top-rhythm and anchor-link-on-hover (¶/#), `scroll-margin-top` for the
  sticky bar.
- **Code blocks per `components/code-blocks.md`** — copy button, language tabs persisting
  globally, command-vs-output discipline. Half of docs quality lives in that module.
- Callouts: 3–4 semantic types max (Note/Warning/Danger + tip), tinted-subtle + icon +
  left border — a page with 6 callouts has buried its prose.
- Every page footer: **Was this helpful? (the feedback loop)** + "Edit this page" (repo
  link) + prev/next pagination within the section.
- Freshness visible: "Updated Mar 2026" — stale-looking docs poison trust in fresh ones.

## Docs-specific behaviors
- Search: instant, grouped by section, shows content snippets with match highlight;
  keyboard-first (`components/search-command.md` at full depth).
- Version switcher when APIs version: the URL carries the version; old-version pages get
  a banner linking current.
- Deep-linkability is sacred: every heading anchored, anchors stable across edits
  (broken anchor links from Stack Overflow answers = lost users).
- Dark mode is table stakes for dev docs — and code themes must switch with it
  (`components/code-blocks.md`).
- API references: sticky endpoint header (verb chip + path), params tables
  (`components/tables.md` alignment rules), request/response examples side-by-side at
  wide widths, in every supported language.

## Landing page of the docs
Not a wall of links: a short "what is this" + THE quickstart CTA + 4–6 top-task cards
("Authenticate", "Send your first event") + search. Route by intent, not by your org chart.

## SEO notes
Docs are the long-tail SEO engine (`checklists/seo.md`): one task per page = one query
per page; titles are the task ("Rotate API keys — <Product> Docs"); breadcrumb structured
data; sitemap complete.

## Anti-patterns
- Marketing voice in docs ("blazingly simple!") — docs speak plainly or lose trust
- One 4,000-word page with all four content types interleaved
- Autogenerated reference AS the only docs (reference without guides is a parts catalog)
- Screenshot-heavy docs of a fast-changing UI (they rot in weeks — crop tightly, show
  states not whole screens, or use live embeds)
- Sidebar mirroring internal team structure instead of user tasks
- Unversioned breaking changes ("the docs are right for SOME version of this")

## Checklist
- [ ] Three-panel shell; ⌘K search; scrollspy TOC
- [ ] Four content types separated; quickstart ≤10 min and polished hardest
- [ ] Code blocks module fully applied; callouts ≤4 types
- [ ] Anchors stable, freshness shown, feedback + edit links
- [ ] Dark mode incl. code themes; version story if applicable
- [ ] One task per page; docs sitemap in SEO gate
