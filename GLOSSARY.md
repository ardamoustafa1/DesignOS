# Glossary

The system's vocabulary, A–Z — each term as DesignOS uses it, with the module that
owns it. One term per concept is law (`agents/copywriter.md`); this file is the law's
dictionary.

**Accent** — the single hue that owns interactivity; punctuation, not prose. `foundations/colors.md`
**Aha-path** — the shortest route from signup to first delivered value. `patterns/onboarding-auth.md`
**Anchor (pricing)** — the honestly-presented higher reference that makes the target tier feel reasonable. `psychology/persuasion.md`
**Anti-pattern** — a recognizable, recurring failure mode; cataloged in the `museum/`.
**Attention audit** — checking that the 3 visually heaviest elements are the 3 most important. `psychology/attention.md`
**Bento** — mixed-size card grid with one hero cell; flattens hierarchy, so only for peer content. `foundations/grids.md`
**Boot sequence** — the kernel's 8 mandatory steps from brief to memory-write. `CLAUDE.md`
**Character preset** — the product's one motion personality (precise/confident/friendly/calm). `motion/principles.md`
**Compositor rule** — animate only transform/opacity; layout properties never tween. `motion/performance.md`
**Doubt peak** — the flow moment where anxiety maximizes (pricing reveal, card form); where reassurance lives. `psychology/persuasion.md`
**Drift** — accumulated off-system values (magic numbers, rogue hexes); measured by `validators/check-drift.js`, reversed by the refactor loop.
**Eye path** — the declared fixation order (1st→2nd→3rd→action) that layout must enforce. `brain/design-intelligence.md`
**Eyebrow** — the small caps label above a heading; the section's category anchor. `foundations/typography.md`
**Fold, complete thought at the** — identity + claim + action visible at 1440 AND 390 before scrolling. `foundations/layout.md`
**Goal-gradient** — motivation rises near completion; why progress starts pre-seeded. `psychology/cognition.md`
**Halation** — pure-white-on-black text bloom that strains astigmatic eyes; why body text is never #FFF. `foundations/dark-mode.md`
**Headline stack** — all headings read alone; must carry the page's complete argument. `patterns/landing-pages.md`
**Instant-fail** — a defect that caps its score dimension at 60 regardless of other merits (contrast failure, fake data, missing keyboard path…). `brain/quality-bar.md`
**Investment mechanic** — user-created state that raises the product's value honestly; lock-in through value, never hostage-taking. `psychology/habit-retention.md`
**Kernel** — `CLAUDE.md`; the boot sequence, routing table, standards, and output contract.
**Layer-cake scan** — real reading behavior: headings + bold + images only; design for the skim. `psychology/attention.md`
**Loop, the** — Research→Wireframe→UI→Review→A11y→Perf→SEO→Refactor→Score; no stage skipped. `loops/design-loop.md`
**Measure** — line length in characters; 60–75ch for prose. `foundations/typography.md`
**Memory** — the seven per-project files that make decision #40 consistent with #4. `memory/README.md`
**Mercy mechanic** — streak freezes, safe backs, save-and-resume; forgiveness as a feature. `psychology/gamification.md`, `components/steppers-wizards.md`
**Module anatomy** — the six-part skeleton every knowledge file follows (principles → specs → patterns → anti-patterns → references → checklist). `ARCHITECTURE.md`
**One-reader sentence** — what the target persona must think to convert ("this would have saved us during…"). `memory/templates/client.md`
**Origin honesty** — elements enter from where they conceptually live (dropdown from trigger, drawer from edge). `motion/principles.md`
**Protected list** — what the creative director marks "don't lose this" across revisions. `agents/creative-director.md`
**Proximity encodes relationship** — related = close, unrelated = far; spacing IS information architecture. `foundations/spacing.md`
**Recognition over recall** — never make users remember across screens. `psychology/cognition.md`
**Risk reversal** — the anxiety-reducing line at the highest-commitment CTA ("Free 14 days · No credit card"). `psychology/persuasion.md`
**Routing table** — the kernel's task-signal → modules map; how lazy loading works. `CLAUDE.md`
**Scrim** — the darkening layer that makes text-on-photo pass contrast at the worst pixel. `foundations/iconography.md`
**Semantic token** — a Tier-2 variable named by meaning (`--bg-surface`), never appearance (`--light-gray`). `foundations/design-tokens.md`
**Signature (moment/vehicle)** — the 1–2 deliberate boldnesses that make the design unmistakably this brand's. `brain/originality.md`
**Squint test** — blurred, the page still shows structure: dominant mass, supporting mass, action point. `brain/quality-bar.md`
**State matrix** — loading/empty×3/partial/error/success — declared complete before review. `components/states.md`
**Swap test** — logo swapped: does the design still belong to this brand? Fail = template. `brain/originality.md`
**Tabular-nums** — equal-width numerals; mandatory wherever numbers align or update. `foundations/typography.md`
**Three timescales** — the squint (0.5s), the glance (3s), the minute (60s) — every screen must win all three. `brain/quality-bar.md`
**Tier (token)** — primitive → semantic → component; components consume semantic only. `foundations/design-tokens.md`
**Trust ledger** — every element deposits or withdraws; remove withdrawals before adding deposits. `psychology/trust.md`
**Whitespace budget** — emptiness spent deliberately as emphasis, luxury, and focus. `foundations/spacing.md`
**Zero-CLS swap** — skeleton→content at identical dimensions; loading never shifts layout. `components/states.md`
