01 Ask in your words

Plain-language questions over the accounts you own. No prompt engineering — the copilot asks a clarifying question when your ask is ambiguous instead of guessing.

02 It reads, it never acts

Analysis runs read-only. Any action — an email draft, a CRM field change — lands in your review queue as a proposal with a diff, never as a done deed.

03 Answers carry receipts

Every claim links to the note, ticket, or invoice behind it. Low-confidence findings are labeled and demoted to "worth checking," not stated as fact.

Great at some things. Deliberately not built for others.

Stated limits are the fastest way to earn trust in a sector that oversells. Here are ours, plainly.

What it does well

  • Renewal-risk and account-health summaries across hundreds of accounts
  • Draft follow-ups grounded in the actual conversation history
  • "What changed since I last looked" digests per account list
  • Flagging contradictions between notes and billing reality

What it refuses to do

  • Send anything externally without a named human approving that send
  • Forecast revenue — that's your model's job; we surface inputs, not prophecy
  • Answer from data you haven't connected (it says "not connected," not a guess)
  • Train on your data — analysis is inference-only, per contract, at every tier

In your stack before the meeting ends

OAuth to your CRM, or the API if you'd rather own the pipes. Both paths are documented end to end.

terminal
npm install @relayer/sdk

import { Relayer } from "@relayer/sdk";

const run = await new Relayer({ apiKey }).analyze({
  scope: "renewals:q3-enterprise",
  sources: ["crm.notes", "billing.events"],   // read-only scopes
  actions: "propose"                          // never "auto"
});

The actions: "propose" default is enforced server-side — an integrator cannot accidentally ship an auto-acting copilot. Read the security model →
