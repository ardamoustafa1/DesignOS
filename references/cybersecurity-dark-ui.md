# Reference Pack: Cybersecurity Dark UI

Use for EDR, SIEM, SOC, GRC, threat intelligence, and security developer tools.

## Taste Signature

- Dark-first, technical, skeptical, precise.
- Monospace texture used for data and identifiers, not body copy.
- Cyan/green/amber accents with restraint.
- Trust comes from architecture, auditability, and concrete limits.

## Transferable Moves

- Show detection logic, alert flow, or investigation workflow.
- Replace fake fear language with specific risk reduction.
- Surface compliance honestly: "evidence workflow" beats fake certification.
- Prefer tables, timelines, and rule mappings over generic feature cards.
- Use dense dashboards for expert users; avoid landing-page spacing in SOC views.

## Do Not Copy

- Hacker movie neon.
- Red alert spam.
- Fake enterprise logos.
- Unsourced MITRE/SOC2 claims.

## Checklist

- [ ] Claims are inspectable or clearly fictional
- [ ] Dark mode contrast is verified
- [ ] Alerts/status use text plus color
- [ ] Pricing explains unit economics
- [ ] Security proof is architecture-first, not testimonial-first
