# Reference Pack: Vercel Docs Style

Use for developer docs, API portals, SDK guides, and technical onboarding.

## Taste Signature

- Fast orientation before deep explanation.
- Strong navigation, narrow prose, copyable examples.
- Versioning and platform constraints visible.
- Minimal visual noise; code is the hero.

## Transferable Moves

- Put the first successful command early.
- Keep examples complete and copy-pasteable.
- Use side navigation plus local table of contents.
- Separate concepts, quickstart, API, and examples.
- Make failure paths easy to find.

## Do Not Copy

- Brand-specific black/white minimalism as a shortcut.
- Docs pages with no working examples.
- API references without conceptual framing.

## Checklist

- [ ] Quickstart reaches success fast
- [ ] Code blocks have language, copy affordance, and expected output
- [ ] Navigation survives mobile
- [ ] Version/current-state is clear
- [ ] Error/debugging section exists
