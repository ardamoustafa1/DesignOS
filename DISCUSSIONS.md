# GitHub Discussions Setup Guide

> **One-time setup required** (repo owner only):
> GitHub repo → Settings → Features → check **Discussions** → Save.

## Recommended categories to create after enabling

| Category | Type | Purpose |
|---|---|---|
| **Q&A** | Q&A | "How do I use DesignOS with Cursor?" — gets an answer |
| **Show and Tell** | Open-ended | Pages built with the system — the living showcase |
| **Ideas** | Open-ended | Module proposals that don't warrant a full issue yet |
| **General** | Open-ended | Everything else |

## Welcome pinned post (paste this as your first Discussion)

> **Welcome to DesignOS Discussions 👋**
>
> This is the place for questions, showcasing what you've built, and proposing new modules.
>
> **Quick links:**
> - Install: `npx github:ardamoustafa1/DesignOS init --agents --skills`
> - Docs: [GETTING-STARTED.md](../GETTING-STARTED.md) · [CHEATSHEET.md](../CHEATSHEET.md)
> - Show your work: [SHOWCASE.md](../SHOWCASE.md) (PR welcome)
> - Propose a module: open an [Issue](../issues/new?template=new-module.md) with the outline
>
> **Rules:** same as the system — every opinion earns its place with reasoning.
> "I prefer X" without a why gets redirected. Bring evidence, get a response.

## Discord alternative

If a Discord server is preferred over Discussions, create one and add the invite link
to `README.md`'s Community section and pin it here. Discord works better for real-time
support; Discussions works better for permanent searchable knowledge. For a dev tool,
Discussions compounds better over time.
