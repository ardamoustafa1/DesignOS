## 問題

AIエージェントは完璧なコードを書けるのに、UIは2015年のまま——窮屈な余白、競合する5つのCTA、グレーonグレーのコントラスト失敗。足りないのは能力ではなく、**審美眼・プロセス・品質ゲート**です。

## 答え

DesignOSはプロンプト集ではなく、エージェントが起動する**オペレーティングシステム**です：

```
入力：      「StripeレベルのSaaSランディングページを作って」

DesignOS：  カーネル起動 → 必要なモジュールへルーティング → 業界ルールを読込
            → デザインループ実行 → 6次元で自己採点
            → 95点未満はすべてやり直し → 決定をメモリへ記録
```

エージェントは「ボタンを作ろう」ではなく、「視線はどこに落ちる？CTAは勝っている？
Appleならこの要素を削るのでは？」と考え始めます。

## 5つのレイヤー

| レイヤー | 役割 | 場所 |
|---|---|---|
| **1 · 知識** | 60超の明確な意見を持つモジュール——基礎、コンポーネント20、心理学8、モーション、パターン12 | `foundations/` `components/` `psychology/` `motion/` `patterns/` `native/` |
| **2 · ループ** | 調査 → ワイヤーフレーム → UI → レビュー → a11y → 性能 → SEO → リファクタ → 採点 | `loops/` |
| **3 · レビューエンジン** | 敵対的な自己採点、6次元、95点の固いしきい値 | `scoring/` + `validators/` |
| **4 · メモリ** | プロジェクトごとに7ファイル：クライアント、ブランド、デザイン、ページ、TODO、バグ、ノート | `memory/` |
| **5 · 業界知能** | 24業種のプレイブック、それぞれ固有のビジュアル言語 | `industries/` |

さらに**9人のエージェントスタジオ**（`agents/`）：クリエイティブディレクター、UXリサーチャー、UIデザイナー、フロントエンドエンジニア、モーションデザイナー、アクセシビリティ専門家、コピーライター、SEO、そして拒否権を持つ敵対的レビュアー。

## クイックスタート

プロジェクトのディレクトリで1コマンド：

```bash
npx github:ardamoustafa1/DesignOS init --agents --skills
```

システムが `./DesignOS` にコピーされ、`CLAUDE.md` に接続され、9人の専門家が本物の
Claude Codeサブエージェントとして登録されます。あとは依頼するだけ：

> *サイバーセキュリティSaaSの料金ページをデザインして。ダークテーマで。*

## 百聞は一見にしかず

[`examples/`](examples/README.md) ギャラリーには、システムが生成した4つの完全なページ
——ランディング、ダッシュボード、料金、ドキュメント——が単一のトークンシステム上に
収録されています。意思決定の記録は[ウォークスルー](examples/saas-landing-walkthrough.md)に。
