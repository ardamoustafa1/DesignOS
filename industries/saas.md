# Industry: SaaS (B2B Software)

## Buyer psychology
Two audiences on every page: the **end user** (will this make my week easier?) and the
**buyer/approver** (is this safe to champion internally?). Design for the user, arm the
champion: shareable pricing, security page, ROI framing, comparison content.
Evaluation is comparative — assume 3 competitor tabs are open. Differentiation must be
visible in 5 seconds (the screenshot does this, not the slogan).

## Visual language
- **Color:** one confident accent on disciplined neutrals; electric blue/indigo are safe-crowded — differentiate via execution or a warm counter-move (`psychology/color-psychology.md`). Light theme default; dark acceptable for technical audiences.
- **Type:** grotesque sans, tight display tracking; the "modern SaaS look" is 80% typography discipline (`foundations/typography.md`).
- **Imagery:** REAL product screenshots, framed, retina — the #1 credibility asset. Abstract illustration only as support. If the UI is ugly, fix the UI, don't hide it.
- **Layout:** split hero or product-first; bento for breadth; sticky tour for workflow depth.

## Page priorities
1. Landing (`patterns/landing-pages.md`) with product visible above fold
2. Pricing — public, self-serve (`patterns/pricing.md`)
3. Integrations page (logo grid + search — ecosystem = adoption safety)
4. Security/trust page (SOC2, GDPR, DPA downloads) — procurement reads this
5. Changelog — visible momentum is a retention & sales asset (Linear doctrine)

## Trust requirements
Customer logos (recognizable > many) · case study with a number · uptime/status page link ·
transparent pricing · migration/import story ("switch from X in a day").

## Motion character
Precise/technical or confident/premium presets (`motion/principles.md`). The site's speed
*is* the product demo — Lighthouse 95+ non-negotiable here of all places.

## In-product notes
Dashboard doctrine applies (`components/dashboard.md`); onboarding time-to-value <5 min
(`patterns/onboarding-auth.md`); empty states = onboarding system.

## Anti-patterns
Feature-noun headlines ("Workflow Automation Platform") over outcome claims · fake dashboard
mockups with lorem metrics · 6 equal feature cards · gradient-mesh sameness ("every SaaS 2024") ·
hiding pricing on a self-serve product.
