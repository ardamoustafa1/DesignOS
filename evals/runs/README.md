# Eval Runs

Living benchmark reports. `evals/RESULTS.md` summarizes the headline table; this folder
holds the detailed raw run reports.

## Naming

- `run-003.md`
- `run-004.md`
- `run-005.md`

Use `evals/RUN_TEMPLATE.md` for each run.

## Acceptance Bar

A run is useful when it includes:

- runner identity or GitHub handle
- exact model and agent surface
- control and treatment setup
- raw generated outputs or links
- validator logs
- blind judge prompt/transcripts
- negative or ambiguous findings

Maintainer-run audits are allowed, but they must be labeled maintainer-run. The
credibility unlock is third-party runs.
