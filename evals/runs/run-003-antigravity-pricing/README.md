## First Deterministic Review

Antigravity produced a polished dark cybersecurity pricing page and a confident
self-score. The deterministic review disagreed:

```text
Overall gate: 75/100
Findings: 7
UI Craft: 75
Conversion: 76
```

Main failures:

- fake customer/logo proof such as "Trusted by 1,847 security teams"
- named logos without evidence
- synthetic testimonials and metrics
- raw SVG/token color drift
- one off-grid spacing value

## Refactor Prompt

The repair prompt asked Antigravity to:

- remove fake customer logos and "Trusted by" copy
- replace proof with fictional/neutral evaluation language
- use tokenized/currentColor SVG values
- align spacing to the 4px grid
- stop claiming "zero raw hex" until review confirmed it

## Second Deterministic Review

After refactor:

```text
Overall gate: 90/100
Findings: 2
UI Craft: 90
Conversion: 100
```

Remaining findings were token false positives:

- `meta name="theme-color" content="#0B0D10"`
- `:root { --border-focus: #22D3EE; }`

This run directly led to the next DesignOS review-engine improvements:

- `:root` token definitions are allowed.
- theme-color meta values are allowed.
- `--fix-prompt` generates an agent-ready repair prompt.
- proof-risk detection now catches certification claims, logo proof, and testimonial
  blockquotes more aggressively.

## Product Lessons

1. Agent self-scores are useful notes, not gates.
2. The strongest differentiator is proof honesty, not visual polish.
3. Repair prompts should be generated by the deterministic tool, not hand-written every
   time.
4. Antigravity needs its own adapter because users split work between terminal and chat.
