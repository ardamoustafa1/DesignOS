# Social Threads

## English thread

1. I built DesignOS: an open-source design brain for AI coding agents. Claude, Cursor,
Copilot, Windsurf, Cline, Aider — any agent that reads markdown can use it.

2. The problem: agents write increasingly good code, then wrap it in weak UI. Poor
hierarchy. Fake testimonials. Low contrast. Five CTAs. No memory of why a design
decision was made.

3. DesignOS gives the agent a process: boot kernel → route modules → load memory →
design loop → adversarial review → accessibility/perf checks → scorecard → redo
anything under 95.

4. It is not a component library. It is judgment as version-controlled markdown:
60+ modules, 24 industry playbooks, 9 specialist agents, 7 memory files, zero runtime
dependencies.

5. The anti-pattern museum is my favorite part: 40+ design mistakes cataloged with
the rule that prevents each one.

6. Honest proof status: current evals are maintainer-run validator checks. They show
the instrumentation works, not that the product is independently proven. Run 003 is
open for external users.

## Turkish thread

1. DesignOS'u yaptım: AI kodlama ajanları için açık kaynak bir "tasarım zekası"
katmanı. Claude, Cursor, Copilot, Windsurf, Cline, Aider — markdown okuyabilen her
ajana çalışıyor.

2. Problem şu: ajanlar kodu iyi yazıyor ama arayüz tarafında hâlâ zayıf kalıyor.
Hiyerarşi kaçıyor, kontrast düşüyor, sahte sosyal kanıt uyduruluyor, kararlar sonraki
oturumda unutuluyor.

3. DesignOS ajana süreç veriyor: kernel boot → doğru modülleri yükle → proje belleğini
oku → tasarım döngüsü → adversarial review → a11y/perf kontrolü → skor kartı → 95 altı
olan her şeyi yeniden yap.

4. Bu bir component library değil. Tasarım yargısını markdown olarak versiyonlanabilir
hale getiriyor: 60+ modül, 24 sektör playbook'u, 9 uzman ajan, 7 dosyalık proje belleği,
sıfır runtime bağımlılığı.

5. En sevdiğim kısım Anti-Pattern Museum: 40+ tasarım hatası, her birinin neden kötü
olduğu ve hangi kuralın bunu engellediğiyle birlikte.

6. Dürüst not: mevcut eval'lar maintainer-run sanity check. Yani araçların çalıştığını
kanıtlıyor; bağımsız dış doğrulama için Run 003 hâlâ açık.
