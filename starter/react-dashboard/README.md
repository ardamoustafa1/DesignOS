# Starter: React Dashboard

Use this for dense app interiors: admin panels, operations views, analytics consoles,
and internal tools.

## First Brief

```text
Design a React dashboard for repeat operational users. Load DesignOS components/dashboard.md,
components/tables.md, components/charts.md, components/navigation.md, components/states.md,
checklists/accessibility.md, and scoring/rubric.md. Prioritize scan path, status clarity,
and complete loading/empty/error/disabled/success states. End with a scorecard and redo
anything under 95.
```

## Required Screens

- Overview
- Detail table
- Empty state
- Error/retry state
- Settings or filters panel

## Verification

```bash
node DesignOS/bin/designos.js review src --no-fail
```
