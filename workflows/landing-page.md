# Workflow: Landing Page Sprint

The most requested DesignOS artifact, end to end. Trigger: "design a landing page for X".

## 1. The five answers (intake minimum)
1. Product in one sentence? 2. Who converts (the one-reader sentence)? 3. The ONE conversion
action? 4. What proof exists (real logos/numbers/quotes — inventory now, placeholders get
flagged from birth)? 5. Sector? (→ `industries/` file, which sets theme, tone, trust
requirements, motion license.)
Missing answers → 70%-rule assumptions, stated in the deliverable.

## 2. Argument before layout
Draft the argument spine as HEADLINES ONLY (`patterns/landing-pages.md` +
`agents/copywriter.md` stack coherence): claim → mechanism (3 sub-claims) → proof → objections
→ ask. The layer-cake test applies to this skeleton *before* any visual work — if the
headline stack doesn't argue, no hero treatment will save it.

## 3. Asset-driven composition
Inventory decides (`brain/decision-framework.md`): strong screenshot → split hero;
message-only → editorial; demo-able product → product-first. Choose the feature-section
format (bento / zig-zag / sticky tour) by content shape, section rhythm by
`foundations/spacing.md`, the ONE gradient/signature moment placement.

## 4. Build through the full Design Loop
All nine stages (`loops/design-loop.md`) — this artifact always gets full ceremony because
it's always conversion-critical. Notes specific to landing pages:
- Copy and UI are ONE stage here — they iterate together (headline length changes hero
  composition; don't design lorem boxes and pour words in later).
- Trust elements verified real at stage 3, not discovered fake at stage 9
  (`psychology/persuasion.md` honesty gate).
- SEO stage is mandatory (this page's whole job is being found and converting).
- The scorecard's Conversion dimension gets the deepest pass: CTA cadence, doubt-peak
  coverage, friction audit.

## 5. Deliver
Output Contract complete: the page (production HTML/CSS/JS or the project's stack — all
states, four breakpoints, both-or-declared themes), scorecard ≥95 all six, rationale,
memory writes (`pages.md` row with intent+metric, `todo.md` placeholder flags).

## Post-launch loop (offer it)
Landing pages are hypotheses. Offer the follow-up: instrument the primary metric, revisit
with real data (which becomes RESEARCH input for iteration 2), quarterly review-loop
health checks (`loops/review-loop.md` recurring mode).
